ESP32 Bruce

ESP32 C++ Wi-Fi BLE RF IR RFID
// Bruce.jpg

Overview

Bruce is an open source, offensive security firmware that transforms the sub $10 ESP32-2432S028 Cheap Yellow Display (CYD) into a powerful "Swiss Army knife" for wireless research and penetration testing. This custom build provides a handheld environment comparable to commercial tools like the Flipper Zero, featuring a highly optimized UI tailored for the CYD’s 2.8" touchscreen and integrated GPIO.

The goal was to create a device that is open, hackable, and capable of handling everything from Wi-Fi deauthentication and IR remote cloning to BLE HID attacks in a single, portable binary. By leveraging the ESP32’s dual-core power, the project demonstrates how affordable consumer hardware can be repurposed into a professional grade research tool.

Architecture

The project is built on a modular abstraction layer designed to maximize the ESP32’s performance while maintaining a responsive user interface.

-FreeRTOS Kernel: The firmware operates on a FreeRTOS driven kernel, utilizing a Symmetric Multiprocessing (SMP) model. This allows for deterministic multitasking, ensuring that time critical wireless operations do not interfere with the UI event loop.
-Dual-Core Task Pinning: To prevent "UI lag" during intensive network attacks, the system pins Radio Operations (Core 0) and UI/Rendering (Core 1) to separate Xtensa cores. This isolation ensures the 2.8 touchscreen remains fluid even while the device is actively deauthenticating WiFi clients or sniffing packets.
-NimBLE Stack Optimization: Unlike standard builds that use the memory heavy Bluedroid stack, Bruce utilizes NimBLE (esp-nimble-cpp). This reduces RAM and Flash consumption by approximately 40%, providing the necessary overhead to run the JavaScript interpreter and large attack payloads simultaneously.
-JavaScript Runtime (mquickjs): An integrated MicroQuickJS (mquickjs) interpreter provides a sandboxed environment for executing custom scripts. This allows for complex logic to be written in JS and executed directly from the SD card without requiring a full firmware recompile.
-Hybrid Storage Architecture:LittleFS (On-Flash): Used for core system assets, credentials, and configuration files requiring high speed, wear leveled access, and SD Card (SPI-Driven): Dedicated to high-volume data exfiltration, including WPA/WPA2 handshake captures (EAPOL), PCAP logs, and the JavaScript tool library.

Modules & Capabilities

  • Wi-Fi: Deauth attacks, beacon flooding, Evil Twin AP, packet sniffer with PCAP export to SD card and Pwnagotchi-compatible handshake collection (EAPOL).
  • BLE: Device scanner, Bad BLE (DuckyScripts via Bluetooth HID emulation), advertisement spoofer, Apple/Samsung notification spam
  • IR: Universal remote with signal capture, replay, and a growing community database(requires external IR module)
  • RFID: NFC/RFID Support via PN532: Interfaces with external modules to read, clone, and emulate NFC tags and Amiibos
  • Sub-GHz: OOK/ASK signal capture and replay (requires external CC1101 module)
  • Tools: UART/I2C/SPI terminal, GPIO tester, WiFi password cracker (PMKID), LED control

Custom Additions

Beyond the base Bruce feature set, this build adds a scripting engine that lets users write simple automation scripts stored on the SD card. Scripts can chain tool actions for instance, scan for a Wi-Fi network, deauth it, spin up an Evil Twin, and log credentials all from a single three line script file.

A web-based config UI is also served over a local AP when the device boots in "config mode", making it possible to update settings, upload scripts, and view logs without a serial connection.

Open Source Contributions

Several modules developed for this project have been upstreamed to the main Bruce repository, including the improved IR database parser (Bruce now supports custom IR signals through LittleFS or SD storage using the standard .ir file format popularized by the Flipper Zero community), the PCAP export feature, and the BLE advertisement spoofing module. The scripting engine is currently in review for upstream merge.
